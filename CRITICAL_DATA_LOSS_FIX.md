# 🚨 КРИТИЧЕСКАЯ ПРОБЛЕМА: Потеря данных пользователей

## Проблема

**На Render файловое хранилище НЕ сохраняется между перезапусками!**

- Файл `backend/data/users.json` создается в эфемерной файловой системе
- При перезапуске Render (деплой, обновление, сон) файл **удаляется**
- Все пользователи **теряются**

## Почему это происходит

Render использует **эфемерную файловую систему**:
- Файлы сохраняются только во время работы сервиса
- При перезапуске все файлы удаляются
- Это стандартное поведение для бесплатного тарифа Render

## Решение

### Вариант 1: PostgreSQL (рекомендуется) ✅

**Преимущества:**
- Данные сохраняются навсегда
- Надежное хранилище
- Render предоставляет бесплатный PostgreSQL

**Шаги:**
1. В Render Dashboard → Create New → PostgreSQL
2. Создайте базу данных
3. Получите connection string
4. Обновите код для использования PostgreSQL (есть примеры в `backend/models/User.postgresql.example.ts`)

### Вариант 2: Persistent Disk (платно)

Render предоставляет persistent disk, но это платная функция.

### Вариант 3: Внешнее хранилище

- MongoDB Atlas (бесплатный tier)
- Supabase (бесплатный PostgreSQL)
- Railway PostgreSQL (бесплатный)

## Быстрое решение (временное)

Можно использовать переменные окружения для хранения данных, но это не надежно.

## Что делать СЕЙЧАС

1. **Немедленно:** Настроить PostgreSQL на Render
2. **Обновить код:** Использовать PostgreSQL вместо файлов
3. **Миграция:** Перенести существующие данные (если есть)

## Код для PostgreSQL уже готов!

В проекте есть готовые примеры:
- `backend/models/User.postgresql.example.ts`
- `backend/models/Diagnosis.postgresql.example.ts`
- `backend/migrations/` - SQL миграции

Нужно только:
1. Создать PostgreSQL на Render
2. Получить connection string
3. Заменить файловые модели на PostgreSQL модели
