## Следующие шаги

1. **Установите зависимости:**
   ```bash
   npm install
   cd backend && npm install
   ```

2. **Настройте переменные окружения:**
   - Создайте `.env` в корне проекта
   - Создайте `backend/.env`

3. **Запустите бэкенд локально:**
   ```bash
   cd backend
   npm run dev
   ```

4. **Настройте EAS Update:**
   ```bash
   npm install -g eas-cli
   eas login
   eas update:configure
   ```

5. **Задеплойте на сервер:**
   - Бэкенд: Railway или другой хостинг
   - Фронтенд: Vercel

6. **Создайте первое обновление:**
   ```bash
   npm run update:app
   ```

## Важные замечания

- **Безопасность:** В production обязательно измените `JWT_SECRET` на случайную строку
- **База данных:** Текущая реализация использует файловое хранилище. Для продакшена рекомендуется PostgreSQL
- **CORS:** Настройте CORS на бэкенде для вашего домена
- **Обновления:** OTA обновления работают только в production сборках, не в development режиме

## Поддержка

При возникновении проблем:
1. Проверьте логи на сервере (Railway/Vercel)
2. Проверьте переменные окружения
3. Убедитесь, что бэкенд доступен и отвечает на запросы
4. Проверьте консоль браузера на ошибки
