# 🚀 Деплой бэкенда на Render (бесплатно)

## Пошаговая инструкция

### Шаг 1: Регистрация

1. Откройте https://render.com
2. Нажмите "Get Started for Free"
3. Войдите через **GitHub**
4. Подтвердите доступ к репозиториям

### Шаг 2: Создание Web Service

1. В Dashboard нажмите "New +"
2. Выберите **"Web Service"**
3. Нажмите "Connect account" → выберите GitHub
4. Найдите и выберите репозиторий `Pelby-HoReCa/horeca-diagnosis`
5. Нажмите "Connect"

### Шаг 3: Настройка

Заполните форму:

**Основные настройки:**
- **Name:** `horeca-backend`
- **Region:** Frankfurt (или ближайший к вам)
- **Branch:** `main`
- **Root Directory:** `backend` ⚠️ ВАЖНО!
- **Runtime:** `Node`
- **Build Command:** `npm install && npm run build`
- **Start Command:** `npm start`
- **Plan:** выберите **Free**

**Переменные окружения:**
Прокрутите вниз до "Environment Variables" и добавьте:

```
NODE_ENV=production
CORS_ORIGIN=https://your-app.vercel.app
JWT_SECRET=ваш-случайный-секретный-ключ-минимум-32-символа
```

**Важно:**
- `PORT` не указывайте - Render сам назначает через переменную `PORT`
- Замените `JWT_SECRET` на случайную строку (минимум 32 символа)
- `CORS_ORIGIN` пока оставьте как есть, потом обновите после деплоя фронтенда

### Шаг 4: Создание

1. Прокрутите вниз
2. Нажмите **"Create Web Service"**
3. Render начнет деплой (3-5 минут)

### Шаг 5: Получение URL

После успешного деплоя:
1. В верхней части страницы сервиса будет URL вида: `https://horeca-backend-xxxx.onrender.com`
2. Скопируйте этот URL
3. Он понадобится для настройки фронтенда

### Шаг 6: Проверка

Откройте в браузере: `https://your-backend-url.onrender.com/health`

Должен вернуться: `{"status":"ok","timestamp":"..."}`

## Важно знать

⚠️ **"Сон" сервиса:**
- На бесплатном плане сервис засыпает после 15 минут неактивности
- Первый запрос после пробуждения может занять 30-60 секунд
- Это нормально для бесплатного плана

✅ **Автоматические обновления:**
- При каждом `git push` Render автоматически пересоберет и задеплоит

## Решение проблем

**Ошибка сборки:**
- Проверьте логи: в Dashboard → ваш сервис → Logs
- Убедитесь, что Root Directory указан как `backend`
- Проверьте, что `package.json` есть в папке `backend`

**Сервис не запускается:**
- Проверьте переменные окружения
- Убедитесь, что `Start Command` правильный: `npm start`
- Проверьте логи на ошибки

**CORS ошибки:**
- Обновите `CORS_ORIGIN` в переменных окружения на URL вашего фронтенда
- Перезапустите сервис (Manual Deploy → Clear build cache & deploy)
