# 🚀 Быстрый деплой на Vercel (5 минут)

## Вариант 1: Через веб-интерфейс Vercel (рекомендуется)

### Шаг 1: Подготовка кода
Убедитесь, что код закоммичен в Git:
```bash
git add .
git commit -m "Готово к деплою"
git push
```

### Шаг 2: Деплой на Vercel

1. **Откройте [vercel.com](https://vercel.com)** и войдите (можно через GitHub)

2. **Нажмите "Add New Project"**

3. **Импортируйте репозиторий:**
   - Если репозиторий на GitHub - выберите его
   - Если нет - загрузите код через GitHub или GitLab

4. **Настройте проект:**
   - **Framework Preset:** Other
   - **Root Directory:** `./` (корень проекта)
   - **Build Command:** `npm run web:build`
   - **Output Directory:** `.expo/web-build`
   - **Install Command:** `npm install`

5. **Добавьте переменные окружения:**
   - `EXPO_PUBLIC_API_URL` = `http://localhost:3000` (пока локально, потом замените на URL бэкенда)
   - `EXPO_PUBLIC_USE_SERVER_API` = `false` (пока используем локальное хранилище)

6. **Нажмите "Deploy"**

7. **Дождитесь завершения деплоя** (обычно 2-3 минуты)

8. **Получите ссылку!** Vercel автоматически создаст URL вида:
   ```
   https://horeca-diagnosis-xxxxx.vercel.app
   ```

## Вариант 2: Через Vercel CLI

```bash
# 1. Установите Vercel CLI
npm install -g vercel

# 2. Войдите в аккаунт
vercel login

# 3. Задеплойте
vercel --prod
```

## После деплоя

1. **Скопируйте ссылку** из панели Vercel
2. **Откройте в браузере** - приложение должно работать!
3. **Поделитесь ссылкой** с пользователями

## Важно

- Приложение будет работать в режиме локального хранилища (без сервера)
- Для полноценной работы с сервером нужно также задеплоить бэкенд (см. `QUICK_START.md`)
- После деплоя бэкенда обновите `EXPO_PUBLIC_API_URL` в настройках Vercel

## Обновление приложения

После изменений в коде:
1. Закоммитьте изменения: `git push`
2. Vercel автоматически задеплоит новую версию
3. Или вручную: в панели Vercel → Deployments → Redeploy

## Проблемы?

- Если сборка не проходит - проверьте логи в панели Vercel
- Убедитесь, что `package.json` содержит скрипт `web:build`
- Проверьте, что все зависимости установлены
