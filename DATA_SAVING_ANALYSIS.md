# 📊 Анализ сохранения данных

## ✅ Ответы на ваши вопросы

### 1. Сохраняются ли результаты после прохождения каждого отдельного блока?

**ДА, сохраняются!**

**Где:** `src/screens/BlockQuestionsScreen.tsx`, функция `markBlockAsCompleted()` (строка 196)

**Что происходит:**
1. После завершения блока вычисляется эффективность
2. Блок обновляется в массиве всех блоков
3. **Сохранение в глобальное хранилище:**
   ```typescript
   await AsyncStorage.setItem('diagnosisBlocks', JSON.stringify(updatedBlocks));
   ```
4. **Сохранение в персональное хранилище пользователя (если авторизован):**
   ```typescript
   const userId = await getCurrentUserId();
   if (userId) {
     await saveUserBlocks(userId, updatedBlocks);
   }
   ```

**Результат:** Данные сохраняются **сразу после завершения каждого блока**.

---

### 2. Все ли данные сразу транслируются в "аккаунт" пользователя?

**ДА, транслируются!**

**Что сохраняется в аккаунт пользователя:**

1. **Блоки диагностики:**
   - Ключ: `user_${userId}_diagnosisBlocks`
   - Сохраняется через `saveUserBlocks(userId, blocks)`
   - ✅ Сохраняется после каждого блока

2. **Задачи:**
   - Ключ: `user_${userId}_actionPlanTasks`
   - Сохраняется через `saveUserTasks(userId, tasks)`
   - ✅ Сохраняется при создании/изменении задач

3. **Данные профиля:**
   - Сохраняются через API (`updateUserData`) или локально
   - ✅ Сохраняются при изменении профиля

4. **История диагностики:**
   - Сохраняется через `saveDiagnosisHistory()` (API или локально)
   - ✅ Сохраняется при завершении всех блоков или переходе к результатам

**Важно:** Если пользователь **не авторизован**, данные сохраняются только в глобальное хранилище (`diagnosisBlocks`, `actionPlanTasks`). После авторизации данные мигрируются в персональное хранилище.

---

### 3. Сохранятся ли данные, если пользователь выйдет и зайдет снова?

**ДА, сохранятся и восстановятся!**

**Как работает восстановление:**

#### При входе пользователя:

1. **Авторизация** (`src/utils/api.ts`, функция `login()`):
   - Сохраняется `authToken`, `userId`, `userEmail`
   - Вызывается `migrateUserData(userId)` - миграция данных в персональное хранилище

2. **Загрузка данных на экранах:**

   **DashboardScreen:**
   ```typescript
   if (userId) {
     storedBlocks = await loadUserBlocks(userId);  // Загружает из user_${userId}_diagnosisBlocks
     storedTasks = await loadUserTasks(userId);    // Загружает из user_${userId}_actionPlanTasks
   }
   ```

   **SelfDiagnosisBlocksScreen:**
   ```typescript
   const userBlocks = await loadUserBlocks(userId);  // Загружает блоки пользователя
   ```

   **ActionPlanScreen:**
   ```typescript
   tasksSource = await loadUserTasks(userId);  // Загружает задачи пользователя
   ```

   **ProfileScreen:**
   ```typescript
   const userData = await getUserData();  // Загружает данные профиля через API
   ```

#### При запуске приложения:

**AppWrapper** (`src/components/AppWrapper.tsx`):
- Проверяет `isAuthenticated` и `authToken`
- Если пользователь авторизован → сразу показывает главный экран
- Данные загружаются автоматически при открытии каждого экрана

## ⚠️ Важные моменты

### Двойное сохранение:
- Данные сохраняются **и в глобальное**, **и в персональное** хранилище
- Это сделано для совместимости и миграции данных

### Приоритет загрузки:
1. Сначала пытается загрузить из персонального хранилища (`user_${userId}_...`)
2. Если нет → загружает из глобального (`diagnosisBlocks`, `actionPlanTasks`)
3. Если нет → использует дефолтные значения

### Серверное сохранение:
- Если `USE_SERVER_API=true`, данные также отправляются на сервер через API
- История диагностики сохраняется через `/api/diagnosis/history`
- Профиль пользователя сохраняется через `/api/users/me`
