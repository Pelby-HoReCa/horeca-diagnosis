# 📋 План создания админки

## Функциональность админки

### 1. Авторизация админа
- Отдельный экран входа для админов
- Проверка роли "admin" в бэкенде
- JWT токен с ролью админа

### 2. Управление пользователями
- Список всех пользователей
- Поиск пользователей
- Фильтрация по дате регистрации
- Детальная информация о пользователе
- Удаление пользователей (опционально)

### 3. Просмотр диагностик
- Список всех диагностик
- Фильтрация по пользователю
- Детальная информация о диагностике
- Просмотр блоков и эффективности
- Просмотр задач

### 4. Аналитика и статистика
- Общее количество пользователей
- Количество завершенных диагностик
- Средняя эффективность по всем диагностикам
- Статистика по блокам
- Графики и диаграммы

### 5. Экспорт данных
- Экспорт пользователей в CSV
- Экспорт диагностик в CSV
- Экспорт статистики

## Технологический стек

### Фронтенд админки

- **React** - UI библиотека
- **TypeScript** - типизация
- **Tailwind CSS** - стилизация
- **Recharts** - графики и диаграммы

## API Endpoints для админки

### Нужно создать в бэкенде:

1. **Авторизация админа**
   - `POST /api/admin/login` - вход админа
   - `GET /api/admin/verify` - проверка токена админа

2. **Пользователи**
   - `GET /api/admin/users` - список всех пользователей
   - `GET /api/admin/users/:id` - детали пользователя
   - `DELETE /api/admin/users/:id` - удаление пользователя

3. **Диагностики**
   - `GET /api/admin/diagnosis` - список всех диагностик
   - `GET /api/admin/diagnosis/:id` - детали диагностики
   - `GET /api/admin/diagnosis/user/:userId` - диагностики пользователя

4. **Статистика**
   - `GET /api/admin/stats` - общая статистика
   - `GET /api/admin/stats/blocks` - статистика по блокам
