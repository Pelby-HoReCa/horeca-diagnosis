# Чеклист развертывания приложения

Используйте этот чеклист для пошагового развертывания приложения.

## Подготовка

- [ ] Установлен Node.js (версия 18+)
- [ ] Установлен Git
- [ ] Создан аккаунт на Expo
- [ ] Создан аккаунт на Vercel (для фронтенда)
- [ ] Создан аккаунт на Railway (для бэкенда) или другой хостинг

## Локальная настройка

### Фронтенд
- [ ] Установлены зависимости: `npm install`
- [ ] Установлен `expo-updates`: `npm install expo-updates`
- [ ] Создан файл `.env` с переменными окружения
- [ ] Проверена работа локально: `npm start`

### Бэкенд
- [ ] Установлены зависимости: `cd backend && npm install`
- [ ] Создан файл `backend/.env` с переменными окружения
- [ ] Проверена работа локально: `npm run dev`
- [ ] Бэкенд отвечает на `http://localhost:3000/health`

## Настройка EAS Update

- [ ] Установлен EAS CLI: `npm install -g eas-cli`
- [ ] Выполнен вход: `eas login`
- [ ] Настроен проект: `eas build:configure`
- [ ] Настроены обновления: `eas update:configure`
- [ ] Обновлен `app.json` с настройками `updates` и `runtimeVersion`

## Деплой бэкенда

- [ ] Создан проект на Railway
- [ ] Подключен GitHub репозиторий
- [ ] Выбрана папка `backend`
- [ ] Добавлены переменные окружения:
  - [ ] `PORT` (автоматически)
  - [ ] `NODE_ENV=production`
  - [ ] `CORS_ORIGIN` (URL фронтенда)
  - [ ] `JWT_SECRET` (случайная строка)
  - [ ] `DATABASE_URL` (если используется БД)
- [ ] Проверен деплой
- [ ] Проверен health check: `https://your-api.railway.app/health`
- [ ] Протестирована регистрация через API

## Деплой фронтенда

- [ ] Создан проект на Vercel
- [ ] Подключен GitHub репозиторий
- [ ] Настроены параметры сборки:
  - [ ] Build Command: `npm run web:build`
  - [ ] Output Directory: `.expo/web-build`
- [ ] Добавлены переменные окружения:
  - [ ] `EXPO_PUBLIC_API_URL` (URL бэкенда)
  - [ ] `EXPO_PUBLIC_USE_SERVER_API=true`
- [ ] Проверен деплой
- [ ] Приложение открывается по ссылке
- [ ] Проверена работа PWA (можно установить на домашний экран)

## Интеграция

- [ ] Фронтенд подключается к бэкенду
- [ ] Регистрация работает и сохраняет данные на сервере
- [ ] Авторизация работает
- [ ] Данные пользователя загружаются с сервера
- [ ] История диагностики сохраняется на сервере

## Тестирование обновлений

- [ ] Внесены изменения в код
- [ ] Создано обновление: `npm run update:app`
- [ ] Проверено, что обновление появилось в EAS
- [ ] Перезагружено приложение
- [ ] Проверено, что изменения применились
- [ ] Кнопка "Обновить приложение" работает

## Безопасность

- [ ] Изменен `JWT_SECRET` на случайную строку
- [ ] Настроен CORS только для вашего домена
- [ ] Проверена валидация данных на бэкенде
- [ ] Пароли хешируются (bcrypt)
- [ ] HTTPS включен на всех сервисах

## Мониторинг

- [ ] Настроены логи на бэкенде
- [ ] Проверены логи на Railway
- [ ] Проверены логи на Vercel
- [ ] Настроен мониторинг ошибок (опционально, например Sentry)

## Документация

- [ ] Обновлен README.md
- [ ] Создана документация API (опционально)
- [ ] Задокументированы переменные окружения

## Финальная проверка

- [ ] Приложение работает по ссылке
- [ ] Регистрация работает
- [ ] Авторизация работает
- [ ] Данные сохраняются на сервере
- [ ] Обновления работают
- [ ] PWA работает
- [ ] Нет критических ошибок в консоли

## Готово! 🎉

После выполнения всех пунктов приложение готово к использованию.
