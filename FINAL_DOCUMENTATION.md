## 📊 Сохранение данных

### Что сохраняется

1. **Профиль пользователя:**
   - Email, имя, телефон
   - Данные о проекте
   - Аватар

2. **Блоки диагностики:**
   - Статус завершения
   - Эффективность
   - Ответы на вопросы

3. **Задачи:**
   - Список задач из плана действий
   - Статус выполнения

4. **История диагностик:**
   - Все завершенные диагностики
   - Блоки, задачи, эффективность

### Где сохраняется

- **Серверный режим** (`USE_SERVER_API=true`):
  - Данные на сервере через API
  - Синхронизация между устройствами

- **Локальный режим** (fallback):
  - Данные в AsyncStorage
  - Только на текущем устройстве

### Восстановление данных

При повторном входе:
- ✅ Профиль восстанавливается
- ✅ Блоки диагностики восстанавливаются
- ✅ Задачи восстанавливаются
- ✅ История диагностик восстанавливается

## 🛠️ Технические детали

### API Endpoints

#### Пользователи
- `POST /api/auth/register` - Регистрация
- `POST /api/auth/login` - Вход
- `GET /api/users/me` - Данные пользователя
- `PUT /api/users/me` - Обновление данных
- `DELETE /api/users/me` - Удаление аккаунта

#### Диагностика
- `POST /api/diagnosis/history` - Сохранение истории
- `GET /api/diagnosis/history` - Получение истории
- `GET /api/diagnosis/history/:id` - Конкретная диагностика

#### Админка
- `POST /api/admin/login` - Вход админа
- `GET /api/admin/verify` - Проверка токена
- `GET /api/admin/users` - Все пользователи
- `GET /api/admin/diagnosis` - Все диагностики
- `GET /api/admin/stats` - Статистика

### Переменные окружения

#### Render (бэкенд)
```
PORT=10000
NODE_ENV=production
CORS_ORIGIN=https://horeca-diagnosis-xxxxx.vercel.app
JWT_SECRET=your-secret-key
ADMIN_EMAIL=Sergo1289
ADMIN_PASSWORD=1289
```

#### Vercel (фронтенд)
```
EXPO_PUBLIC_API_URL=https://horeca-backend-6zl1.onrender.com
EXPO_PUBLIC_USE_SERVER_API=true
```

#### Vercel (админка)
```
NEXT_PUBLIC_API_URL=https://horeca-backend-6zl1.onrender.com
```
